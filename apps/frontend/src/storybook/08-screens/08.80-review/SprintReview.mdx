# 08.80 Review — Sprint-Review Screen

Erste Story im **08-Screens**-Tier (GF-2 T11). Die Screen-Komposition: der V2-Review-View-Container
`SprintReviewV2` (→ `ReviewFlow`, 06.80 Deps & Review, `status:stable`) eingebettet in die kanonische
**AppShell** (05.80). Dies ist der reale Screen, den die Route `/:slug/review/:sprintId` nach dem
Cutover (T13) rendert.

## Zweck & Scope

- **Eigener Screen** (D03 — kein Overlay): binäres Verdict (D01), runden-basierter PO-Review-Flow,
  US-Prüfgrundlage mit Info-Overlay, Volltextsuche, Sort, Runden-Filter, Copy-Markdown.
- **Daten pre-Cutover:** gegen die SOLL-Fixture `review-issue.json` (`user_stories[]`-Form). Der Container
  besitzt (noch) **keine eigene fetch-Mutation** — die Verdrahtung gegen die `user_stories[]`-API
  (`us_verdict → verdict`-Mapping an der Datengrenze, BE-B02) + das MSW-Wiring folgen im **Cutover (T13)**.
- **Scoping (D-Q01):** `ReviewFlow` ist reine Orchestrierung → die play-Funktionen prüfen Callback-/
  UI-Verhalten (Suche/Filter) **ohne MSW**. Verdict-Persistenz lebt in `ReviewVerdict`/`ReviewStep`.

## Achsen (Story-Exports)

| Export | Szenario |
|---|---|
| `Default` | Minimal-Props → Einzel-Fixture (kein Crash) |
| `Main` | Realistische gemischte Runde, Default-Filter offen/abgelehnt |
| `State_Empty` | Leere Runde → `EmptyState` im Detail-Pane |
| `Variant_OnlyRejected` | R3-Default-Filter (offen/abgelehnt zuerst, passed ausgeblendet) |
| `Variant_ReReview` | Bereits passed Issue erneut bewertbar (manueller Reopen, DD-662 / D04) |

| `Interaction_Search` | play: Volltext filtert die Master-Liste (key+title) |
| `Interaction_Filter` | play: Runden-Filter open→all macht passed-Issues sichtbar |

## data-ui-Inventar (Screen-Ebene)

- `screen.sprint-review.*` — Screen-Wrapper je Export
- `app-shell.root` / `app-shell.content` — AppShell-Frame (05.80)
- `review-flow.root` / `.page-title` / `.search` / `.complete` / `.copy` / `.empty` — ReviewFlow (06.80)

> **Cutover-Hinweis (T13):** Der C4-`data_ui`-Bestand führt die Review-Anker unter dem Domänen-Prefix
> `sprint-review.*` (D11). Die realisierten Komponenten verwenden `review-flow.*` — beim `draft→active`-Flip
> ist diese Prefix-Konvention zu reconcilen (`validate_architecture_yaml.py`).
