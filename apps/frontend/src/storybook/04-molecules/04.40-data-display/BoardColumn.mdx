# BoardColumn

Die kanonische **Board-Spalten-Hülle** (04.40 Data Display, GF-322). Komponiert
`Card` (tone=mantle) + Header-Cluster + Body-Slot + optionalen Footer + Empty-State.
Geteilt von **RoadmapBoard** (OR-13, Milestone-Spalten) und **SprintBoard** (OR-14,
Status-Lanes) — ersetzt das dort zuvor duplizierte Inline-Column-Chrome.

## Zweck

Ein Board ist N Spalten gleicher Bauart. Die Spalten-Hülle (gerahmte Fläche, Header
mit Titel + Trailing-Info, Item-Liste, Leer-Hinweis) ist die wiederholbare Einheit →
ein Molekül, kein per-Board inline hingeschriebenes Chrome. Das hält beide Boards
konsistent und erfüllt das Inline-Markup-Verbot im Organism-Tier (D03,
`tests/frontend-rework/organism-inline-markup-budget.test.js`).

## Wann

- Eine neue Board-artige Ansicht (Spalten/Lanes mit Kopf + Liste) braucht die Hülle.
- NICHT für einzelne Detail-Cards (→ `Card`) oder Listenzeilen (→ `EntityItem`).

## Props

- `title` — Header-Label (das Molekül setzt uppercase + truncate).
- `headerEnd` — Trailing-Slot rechts im Header (StatusBadge, Count-/Deferred-`Pill`).
- `empty` / `emptyHint` — Empty-State statt `children`.
- `footer` — Slot unter dem Body (z.B. `AssignDropZone` im SprintBoard).
- `data-ui` — **Parent-Scope (D01, Pflicht beim Consumer):** der Consumer übergibt
  `roadmap-board.column-<key>` bzw. `sprint-board.lane-<key>`; das Molekül leitet
  `${scope}.header` / `${scope}.title` / `${scope}.empty-hint` ab.

## Zustände

Gefüllt (Default) · mit Footer (WithFooter) · leer (Empty).

## a11y

Header ist ein echtes `<header>`-Element (Struktur-Semantik gehört ins Molekül, nicht
in den Organism). Trailing-Info ist rein visuell; die Item-Liste trägt die
Adressierbarkeit über den Parent-Scope.

## Header lean (PO 2026-06-18)

Header trägt NUR Titel + `headerEnd` — bewusst **keine** description-Zeile. Die
border-b ist kanonisch (GF-316, „Column-Header klar erkennbar") für beide Boards.
