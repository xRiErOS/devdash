# MarkdownField

`status:review` · Molecule · Cluster `04.10 Form`

## Zweck

Dünner Wrapper um `MarkdownNoteField` aus dem vendored `note-field`-Paket (CodeMirror 6 +
ixora Live-Preview, Catppuccin). `note-field` IST der Editor — diese Molecule hält nur den
stabilen Prop-Vertrag (`value` + `onChange`-String), damit kein Consumer-Aufrufort angefasst
werden muss. `note-field` bringt eine eigene Toolbar inkl. Live-Preview mit.

Consumer: `ContentBlockForm`, `ChecklistDetailModal`, `SprintForm` (Notizen), `SprintReview` —
alle über diese Molecule (kein direkter `MarkdownNoteField`-Import).

## Props

`value` (Markdown-String, controlled) · `onChange(value)` · `rows` (Mindesthöhe → note-field
`minRows`) · `placeholder` · `disabled` (→ `readOnly`) · `onUpload`/`onRemove` (Bild-Hooks).
Die alten Toggle-Props (`mode`/`onModeChange`/`defaultMode`/`previewEmptyLabel`) werden
akzeptiert, aber von note-field selbst gehandhabt (DD-365).

## Theme-Sync (Erkenntnis 2026-06-21)

**Symptom:** Im Dark-Mode blieb der Editor-Block hell (Catppuccin-Latte), während der Rest der
UI dunkel schaltete — gut sichtbar in Storybook/headless.

**Root-Cause:** `note-field` skaliert sein Theme über `data-theme` auf der EIGENEN `.nf-root`
(Prop `theme`, Default `auto`). Die CSS-Regeln sind `.nf-root[data-theme=dark]` bzw.
`.nf-root[data-theme=auto]` / `.nf-root:not([data-theme])` → `@media (prefers-color-scheme)`.
Der Default-`.nf-root{}` ist die **Latte-Palette**. Die App steuert ihr Theme aber per
`data-theme` auf dem `html`-Element — note-field liest das NICHT. Ohne Bridge fällt note-field
auf `prefers-color-scheme` zurück; in Storybook/Playwright (headless = light) heißt das: Latte,
egal welches App-Theme aktiv ist.

**Fix:** MarkdownField spiegelt das App-Theme reaktiv auf die `theme`-Prop — es liest
`document.documentElement` `data-theme` und beobachtet Änderungen via `MutationObserver`
(`attributeFilter: ['data-theme']`). Werte: `dark` / `light` / `auto`.

**Wichtig für Tests:** Der Node-Render (`renderToStaticMarkup`, kein `document`/`useEffect`)
bleibt `auto` = bisheriger note-field-Default → KEINE Snapshot-Drift. Der echte Theme-Wechsel
greift nur clientseitig (Browser/Storybook), wo der Toolbar/Editor jetzt dem Toggle folgt.

**Merksatz:** Vendored-Editoren mit eigenem Theme-Root brauchen eine explizite Theme-Bridge —
`prefers-color-scheme` deckt den App-Theme-Toggle NICHT ab (und maskiert den Bug auf
Dark-OS-Maschinen).
