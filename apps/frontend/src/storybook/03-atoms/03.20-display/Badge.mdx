# Badge

`status:stable` · Atom · Cluster `03.20 Display`

## Zweck

Basis-Atom: farbcodierter Label-Chip (rounded-full) auf der direkten
Catppuccin-Tone-Achse. Trägt KEINE Status-/Domänen-Semantik — das ist die
Variante `StatusBadge`. Props-driven.

## Wann verwenden

- **Ja:** Tone-codiertes Mini-Label (Status, Kategorie) ohne Akzent-Semantik.
- **Nein:** semantischer Akzent-Chip (primary/success/…) → `Pill`. Konkreter
  Status-String → `StatusBadge` (mappt status→tone+label). Entitäts-Kennung →
  `EntityPill`.

## Barrierefreiheit

- Reines Display; die Tone ist redundant zum Label-Text (Farbe nie alleiniger
  Bedeutungsträger).
- Kontrast: Palette-Token gegen `--on-accent` WCAG-AA.
