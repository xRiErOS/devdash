# Checkbox

`status:stable` · Atom · Cluster `03.10 Inputs`

## Zweck

Token-saubere Boolean-Eingabe (Catppuccin). Custom-Box auf nativem
`<input type="checkbox">` (a11y bleibt erhalten) mit Check-/Minus-Icon.
Props-driven, kein Store — Teil der Inputs-Trias (Text/Boolean/Dropdown)
neben `Button`, `Input` und `Select`.

## Wann verwenden

- **Ja:** einzelne An/Aus-Entscheidung oder Mehrfachauswahl in einer Liste.
- **Nein:** sich gegenseitig ausschließende Optionen → `RadioGroup`.
  Sofort wirksamer Modus-Schalter → `Toggle`/`Switch`. Dropdown-Auswahl → `Select`.

## Zustände

`checked` (gefüllte Box + Check-Icon), `indeterminate` (Minus,
`aria-checked="mixed"` — visueller Misch-/Teilauswahl-Zustand), `disabled`
(opacity + `cursor-not-allowed`). `size` (`sm` = 14px, `md` = 16px Box).

## Barrierefreiheit

- Natives `<input type="checkbox">` — Fokus + Space-Toggle gratis.
- `indeterminate` setzt `aria-checked="mixed"`; sonst spiegelt `aria-checked`
  den `checked`-Zustand.
- Sichtbarer Fokusring via `focus-visible:outline` (Akzent-Token).
- Das `<label>` umschließt Box + Text → der Klick auf das Label toggelt.
- Kontrast: Check-/Minus-Icon nutzt `--on-accent` gegen den `--accent-primary`-Fill (WCAG-AA).
