# DetailStatusStepper

`status:stable` · Organism · Cluster `05.50 Status`

## Zweck

Kanonischer Status-Fokus am Kopf der Detail-Ansicht (DD-635). Der aktuelle
Status sitzt mittig in der Primärfarbe; die via `getValidIssueTransitions`
zulässigen Übergänge sitzen gedimmt davor (lifecycle-früher, links) und dahinter
(rechts). Ein Klick auf einen Übergang löst `onSelect(toStatus)` aus — das
Backend (`canTransition`) bleibt autoritativ und rejected Edge-Cases.

Organism-Tier (nicht Atom/Molekül): trägt Interaktion + Transition-Logik
(Achsen-Einsortierung + valide Folgestati), nicht nur die Präsentation eines
Werts.

## Wann verwenden

- **Ja:** Status eines Issues in der Detail-Ansicht zeigen UND ändern — geteilter
  Baustein für Vollbild-Detail (unter 1024px) und Two-Pane (ab 1024px).
- **Nein:** reine Lifecycle-Anzeige ohne Übergänge → `ProgressDots` (Molecule).
  Einzelner Status-Marker → `Badge`/`EntityPill` (Atom).

## Props

<ArgTypes of={DetailStatusStepperStories} />

`current` = aktueller Status (mittig). `onSelect(toStatus)` macht die Übergänge
klickbar; ohne Handler bleiben sie statisch (read-only). `disabled` deaktiviert
die Knöpfe.

## Zustände

Status-Walk (jeder `current` zeigt sein valides Transitions-Set) · interaktiv ·
disabled · read-only (ohne `onSelect`).

## Barrierefreiheit

- Übergänge sind echte `<button type="button">` mit `title` „Status → `<Label>`".
- Der aktuelle Status ist visuell (Primärfarbe) und per Label ausgewiesen.
- Horizontal scrollbar (`overflow-x-auto`) → kein Layout-Bruch bei schmalen
  Viewports (@393).
- Farben folgen den Akzent-Tokens (WCAG-AA), Bedeutung trägt das Label.
