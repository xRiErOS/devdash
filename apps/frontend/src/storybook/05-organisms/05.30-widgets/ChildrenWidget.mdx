# ChildrenWidget

`status:stable` · Organism · Cluster `05.30 Widgets` · OR-22 · GF-257

## Zweck

Listet die Kinder einer Entität als Accordion: **Milestone → Sprints**, **Sprint →
Issues**, **Issue → SubTasks**. Jede Zeile (komponiert `AccordionRow`, GF-240) klappt
ein Panel mit dem Kind-Detail auf. Das Widget hält den Aufklapp-Zustand.

## Ebene-6-Capability-Gegencheck (A4)

OR-22 = 7 Caps: `sprint-read`, `subtask-add`, `subtask-list`, `subtask-update`,
`subtask-done`, `subtask-delete`, `subtask-reorder` (`_ebene6.md`). Die SubTask-Ebene
(IS → SubTasks) deckt die subtask-Verben ab; SP → Issues nutzt `sprint-read`.

## Wann verwenden

- **Ja:** untergeordnete Einträge einer Entität gebündelt + aufklappbar zeigen.
- **Nein:** Pfad nach oben (Vorfahren) → globales Breadcrumb (Hierarchie-Widget entfernt, GF-256-Rückbau).

## Props

<ArgTypes of={ChildrenStories} />

`items` = `{ key, label, status?, panel? }[]`. `childLabel` benennt die Kind-Ebene
(„Sprints"/„Issues"/„SubTasks"), `defaultOpenKey` öffnet eine Zeile initial (Story/SSR;
produktiv intern getoggelt). Nur eine Zeile ist gleichzeitig offen.

## Zustände

Geschlossen · eine Zeile offen (Panel sichtbar) · leer (Hinweis).

## Barrierefreiheit

- Trigger ist ein `<button>` mit `aria-expanded` + `aria-controls` (Panel-`region`).
- Status trägt der `StatusBadge` (Text + Tönung), nicht nur Farbe.
