# EntityItem

`status:stable` · Organism · Cluster `05.10 Lists` · OR-06 · D04

## Zweck

Die kanonische Listen-Zeile/-Karte für **alle Entity-Typen** (Meilenstein/Sprint/
Issue/DoD/ToDo via `entity`). `layout=card|row` × `surface=filled|bare` × `size`
schalten die Darstellung. Die Item-Infos sind ein **Text-Grid** (R3-EI-1, s.u.) +
ein Disclosure-Chevron im Head + Desktop-DragHandle + (im Select-Mode) eine Checkbox.

Organism-Tier: trägt Selektions-/Expand-State + Drag-Interaktion (props-getrieben,
der ListView/Consumer besitzt den State).

## PO-Review Round 3 (R3-EI-1) — Text-Grid statt Pills

**Global**: `EntityPill`, `StatusBadgeSelect` und `ProgressBar` sind **entfernt** —
die Item-Infos werden als schlichte Text-Zellen gerendert („visuell vereinfachen",
PO). Jede Zelle ist 1:1 adressierbar:

| Anker | Inhalt |
|---|---|
| `…id` | ID-Key (mono), z.B. `DD-42` |
| `…type` | Entity-Typ als Text (`issue`/`sprint`/…) |
| `…name` | Haupt-Label (flex, truncate) |
| `…priority` | Priority als Text (optional) |
| `…progress` | Fortschritt als `v/max`-Text (statt Bar) |
| `…status` | Status als Text (statt Badge) |

**Konsequenz (PO-akzeptiert):** die zeilen-interne **interaktive** Status-Änderung
(EI-6) entfällt — Status ist Text. Bulk-Statuswechsel laufen über den
`StatusChangeModal` (GF-290); Einzel-Status in der Detail-Ansicht.

## Drag & Drop (EI-11)

Der DragHandle (`organism.entity-item.drag-handle`, `cursor-grab`) ist der einzige
Anfasser — Drag startet nur dort, nicht auf der ganzen Zeile (verhindert
versehentliches Ziehen beim Klick auf Status/Disclosure). Das **Reorder-Verhalten**
(Drag→Drop→neue Reihenfolge) besitzt der Consumer/ListView (DnD-Kontext, Persistenz
P3); die Story modelliert nur den visuellen `dragging`-Zustand (gedimmt). **Touch-Drag
ist deferred (D09)** — Desktop-only.

## Wann verwenden

- **Ja:** als Item-Baustein einer `ListView` (GF-244) — Karte oder Zeile, je Entity-Typ.
- **Nein:** die ganze Liste → `ListView`. Reine Identitäts-Anzeige → `EntityPill` (Atom).

## Barrierefreiheit

- Select-Checkbox ist eine echte `<input type="checkbox">` (Checkbox-Atom) mit Label;
  Selektion ist über `aria-selected` **und** Ring-Akzent ausgewiesen.
- DragHandle ist ein `IconButton` mit `aria-label` „Ziehen zum Sortieren".
- Disclosure ist ein `IconButton` mit `aria-expanded`; das Panel ist `role="region"`.
- ID/Typ/Status/Progress/Priority sind Klartext-Zellen (R3-EI-1) — Bedeutung trägt der Text selbst (keine reine Farbcodierung mehr).
