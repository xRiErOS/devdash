## Issue — `GET /api/backlog/:id`

Schlüssel-Felder: `key` (`<prefix>-<project_number>`), `status`, `type`, `priority`,
Refinement-Felder (`goal`, `background`, `context_notes`, `relevant_files`), `result`,
`assigned_sprint` + `sprint_key`, `review_status`, sowie nested `tasks` / `subtasks` /
`user_stories` / `feedback` / `dependencies`. (E01/D09: `acceptance_criteria` +
`test_instruction` abgelöst durch `user_stories[].qa`.)

Enums: `ISSUE_TYPES = bug|feature|improvement|core` · `ISSUE_STATUSES = new|refined|planned|in_progress|to_review|passed|rejected|done|cancelled`

<pre>{JSON.stringify(issue, null, 2)}</pre>

## Sprint — `GET /api/sprints/:key`

`key` (`DD#<n>`), `status` (`planning|active|review|completed|closed|cancelled`),
`milestone_id`, `wip_limit`, plus nested `items[]` (volle Backlog-Rows + `review_status`
+ `screenshot_files`).

<pre>{JSON.stringify(sprint, null, 2)}</pre>

## Milestone — `GET /api/milestones/:id`

`status`, `target_date`, `deferred`, nested `sprints[]` (mit `issue_total/done/cancelled`),
`dod_items[]`, `dependencies_in/out[]`, plus aggregierte Counts (`total`, `done`,
`cancelled`, `terminal_count`).

<pre>{JSON.stringify(milestone, null, 2)}</pre>

## DoD-Item — `GET /api/milestones/:milestone_id/dod-items`

Nested in Milestone-Response als `dod_items[]`. CRUD über dedizierte Routes. `done` ist
`0|1` (SQLite integer). `position` bestimmt Reihenfolge (Reorder via PUT).

Contract: `dodItemCreateContract` (label Pflicht, details optional) · `dodItemUpdateContract`
(label/done/details alle optional).

<pre>{JSON.stringify(dodItem, null, 2)}</pre>

## Project — `GET /api/projects/:id`

`prefix` ist Key-Präfix für Issues (`DD2-42`). `slug` = URL-Safe-Name für Capture-API.
`color` = Catppuccin-Token-Name. `archived = 0|1`.

<pre>{JSON.stringify(project, null, 2)}</pre>

## Tag — `GET /api/tags`

`color` ∈ `blue|green|peach|mauve|teal|overlay0`. Tags sind project-scoped (`project_id`).
Vollständiger Replace via `PUT /api/backlog/:id/tags` mit `{ tag_ids: number[] }`.

<pre>{JSON.stringify(tag, null, 2)}</pre>

## UserStory — `GET /api/backlog/:id/user-stories`

`key` generiert als `US-${id}`. `us_verdict` ∈ `open|accepted|rejected` (Story-Ebene,
≠ Issue-`review_status`). `qa`-Feld = Single Source für Prüfkriterien (E01/D09, ersetzt
altes `acceptance_criteria` + `test_instruction` auf Issue-Ebene).

<pre>{JSON.stringify(userStory, null, 2)}</pre>

## Todo — `GET /api/projects/:project_id/todos`

`status` ∈ `open|done|cancelled`. `links[]` typisiert mit `type ∈ spec|issue|vault|url`.
Position via Reorder-Route (`PATCH /api/projects/:pid/todos/reorder`).

<pre>{JSON.stringify(todo, null, 2)}</pre>

## ProjectMemory — `GET /api/project-memories/:id`

`category` ∈ `architecture_decision|dead_end|bug_pattern|convention|external_constraint|session_log|knowledge`.
`importance` 1 (hoch) – 3 (niedrig). `anchor` = unique String-Identifier (optional).
`stability` ∈ `stable|volatile`. Ersatz via `/supersede` (append-only Korrektur-Chain).

<pre>{JSON.stringify(projectMemory, null, 2)}</pre>

## Attachment — `GET /api/backlog/:id/attachments`

Array von Datei-Referenzen. `file_path` ist relativer Pfad im Projekt-`repo_path`.
Upload via `POST /api/backlog/:id/attachments` (multipart oder `{ file_path: string }`).

<pre>{JSON.stringify(attachment, null, 2)}</pre>
