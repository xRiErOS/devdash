# EntityPill

`status:stable` · Atom · Cluster `03.20 Display`

## Zweck

Kanonische Entitäts-Pille — EINE Wahrheit für ID-Kennungen von Sprint, Issue,
Milestone, Definition-of-Done und ToDo (konsolidiert die früheren
SprintPill/IssuePill/MilestonePill, D-pill-badge-taxonomy). Props-driven, kein Store.

## Wann verwenden

- **Ja:** kompakte Anzeige einer Entitäts-Kennung (`DD-42`), optional mit Name,
  als Label, Link oder klickbarer Button.
- **Nein:** generischer Farb-Chip ohne Entität → `Pill`. Status-Codierung →
  `StatusBadge`/`Badge`. Interaktiver Status-Wechsler → `StatusBadgeSelect` (Organism).

## Barrierefreiheit

- `aria-label` trägt den vollen Kontext (`<id> — <name>`); `title` als Tooltip.
- Icon + Separator sind dekorativ (`aria-hidden`); ID/Name tragen die Bedeutung.
- Button-Modus: natives `<button type>`, Klick/PointerDown stoppen Propagation
  (drag-sicher in dnd-Listen). Link-Modus: natives `<a href>`.
- Kontrast: Akzent-Tokens WCAG-AA.
