# IconButton

`status:stable` · Atom · Cluster `03.10 Inputs`

## Zweck

Icon-only Action-Primitive mit garantiertem Touch-Target. Trägt eine klickbare
Aktion, die rein durch ihr Icon kommuniziert (kein sichtbarer Text). Props-driven,
kein Store — Leaf-Atom, komponierbar in Toolbars/Molecules.

## Wann verwenden

- **Ja:** dichte Aktions-Cluster (Toolbar, Tabellen-Zeile, Drawer-Header), wo
  Text-Buttons zu schwer wären — Löschen, Bearbeiten, Kopieren, Schließen.
- **Nein:** primäre Aktion mit Text → `Button`. Navigation → `Link`.
  Toggle/Mehrfachauswahl → `TabButton` / `Checkbox`.

## Barrierefreiheit

- Natives `<button type="button">` — Fokus + Enter/Space gratis.
- `label` ist **Pflicht** und setzt `aria-label` + `title` — ohne sichtbaren
  Text trägt das Label die gesamte Bedeutung.
- Das Icon selbst ist `aria-hidden` (über `<Icon>` ohne `label`-Prop).
- `disabled` blockt Fokus/Klick; min. 44px (`lg`) für Mobile-Touch.
