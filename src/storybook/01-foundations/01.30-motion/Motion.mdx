# Motion

`status:stable` · Foundation · Cluster `01.30 Motion`

## Zweck

Motion-Tokens (`src/index.css`, D02-Spec) für konsistente Animationen im Developer
Dashboard. Drei Dauer-Stufen (`fast`/`base`/`slow`) + zwei Easing-Kurven
(`ease-standard`/`ease-emphasized`) als CSS-Variablen. Interaktions-States
(Press-`scale` + Focus-Ring) sind über Token-Klassen definiert — 0 inline-`style`.

Spec-Quelle: project_memory D-motion-spec (mem 395).

## Wann verwenden

- **fast (120ms):** Micro-Feedback (Hover, Focus-Ring).
- **base (180ms):** Standard-Transitions (Farb-Wechsel, Opacity).
- **slow (280ms):** Einblenden von Elementen, Modal-Open.
- **ease-standard:** dezeleriert — für ausgehende/erweiternde Animationen.
- **ease-emphasized:** betont — für eingehende/kollabierungsartige Animationen.
- **`prefers-reduced-motion`:** alle Token-Transitions werden auf 0ms reduziert (a11y,
  native CSS `@media (prefers-reduced-motion: reduce)`).

## Barrierefreiheit

- Alle Transitions respektieren `prefers-reduced-motion: reduce` (CSS-native).
- Focus-Ring (`--focus-ring`, Größe l) ist obligatorisch für Tastatur-Fokus — nie
  unterdrücken oder auf 0 setzen.
- Animationsdauer unter 200ms gilt als nicht-ablenkend (WCAG 2.3.3).
