# AppShell — Screen (08.10 Shell)

**Kurzbeschreibung**
Der kanonische, verdrahtete App-Frame (Top-Leiste + Icon-Rail + Content-Outlet) als komponierter
Screen. Rail navigiert real (MemoryRouter), Topbar-Controls feuern, Suche öffnet ein IssueDetails —
alles play-getrieben abgesichert (Behavioral-Checkliste Level-2). Dies ist der LIVE-Shell-Stand,
kein Draft-Ziel mehr.

## Aktueller Stand

### Rail (`app-shell.rail`)
- 4 Quick-Access-Items (Home · Roadmap · Backlog · Memories), Icons via Registry, route-gebunden.
- Wiring-Stand: verdrahtet. Klick navigiert (MemoryRouter), aktiver State folgt der Route. 2 States
  (marker ↔ icon-only) via Toggle unten-rechts (`app-shell.rail.toggle`).

### Topbar (`app-shell.topbar`)
- Quick-Switcher-Trigger = DD-Logo → `ProjectQuickSwitcher`-Panel; Projektwahl wechselt die Breadcrumb-Wurzel.
- Breadcrumb route-abgeleitet, bewusst ≤2 Segmente (Molecule emittiert `breadcrumb.separator` je Segment).
- Suche (Ghost) → leichte Ergebnis-Liste (Mock) → Klick navigiert zu IssueDetails im Content-Outlet.
- Theme-Toggle (Sun/Moon) verdrahtet. Shortcuts-Trigger present.
- Wiring-Stand: alle Controls verdrahtet (play-belegt).

## Verwendung

### Wann verwenden
Als globaler Rahmen jedes produktiven App-Screens (Rail + Topbar + Content-Outlet) plus als
play-Referenz für die Shell-Navigation (Rail / Quick-Switcher / Suche).

### Wann nicht verwenden
Capture-Views (eigener Host, kein Rail/Topbar). Kein produktiver Such-Ergebnis-Screen — die
Ergebnis-Liste hier ist die Navigations-Demo (greenfield Such-Screen ist eigener SOLL, out-of-scope).

## Interaktionen (play, Level-2)

- `Interaction · Controls verdrahtet` — Theme-Toggle + Rail-Collapse feuern.
- `Interaction · Rail-Navigation` — Rail-Klick wechselt View UND Breadcrumb (route-getrieben).
- `Interaction · Quick-Switcher öffnen + Projekt wählen` — Panel öffnet, Projektwahl wechselt die Breadcrumb-Wurzel.
- `Interaction · Suche → Ergebnis → IssueDetails` — Eingabe filtert, Klick navigiert ins Detail.

Lauf: `npm run test:storybook` (CT-Runner via Playwright). Play-Tests sind ungated (Bonus-QA), nicht im `tests/frontend-rework`-Gate.

## Barrierefreiheit

### ARIA
Rail = `role="navigation"`, Topbar = `role="banner"`, Content = `role="main"`. Aktives Rail-Item via
Wrapper-Akzent; Breadcrumb-Aktiv via `aria-current="page"`. (Prüfung: Story-Board-Checkliste.)

### Keyboard
Suche fokussierbar; Controls per Tab erreichbar. (Verhalten: Behavioral-Checkliste Level-2.)

## Kanonische States

### Rail eingeklappt / aufgeklappt
icon-only ↔ marker-Rail via Toggle — `Variant · Rail collapsed`.

### Navigation / Detail
Route-getriebener View-Wechsel (Rail) und Detail-Aufruf (Suche → IssueDetails) — `Interaction_`-Stories.
