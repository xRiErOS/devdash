# StatusBadgeSelect

`status:stable` · Organism · Cluster `05.20 Actions` · GF-286 (Q01)

## Zweck

Vereint **StatusBadge** (Anzeige) und ein **Auswahl-Menü** zu einer interaktiven
Funktion: das aktuelle Status-Badge ist der Trigger; ein Klick öffnet ein Menü
**nur der erlaubten Übergänge** (`allowedStatuses`). Auswahl → `onChange(status)`.

Löst die vom PO gemeldete Redundanz (EI-6): in `EntityItem` ersetzt
StatusBadgeSelect das Paar „statisches StatusBadge + separates StatusPillSelect"
durch ein klickbares Badge, das nur valide Transitions anbietet.

`StatusPillSelect` (GF-231, native Select, `stable`) bleibt unverändert für reine
Formular-Kontexte; StatusBadgeSelect ist die Organism-Variante mit Badge-Trigger +
Transitions-Gate.

## Wann verwenden

- **Ja:** Status eines Items in einer Liste/Detail-Zeile anzeigen UND per Klick
  auf nur valide Folgestati wechseln.
- **Nein:** reine Anzeige ohne Wechsel → `StatusBadge` (Atom). Vollständiger
  Lifecycle-Walk mit Vorher/Nachher → `DetailStatusStepper` (Organism).

## Props

<ArgTypes of={StatusBadgeSelectStories} />

`allowedStatuses` = `string[]` oder `{value,label?}[]` — der Consumer liefert nur
die Lifecycle-validen Ziele (Backend bleibt autoritativ). `defaultOpen` steuert den
Initialzustand (für Story/SSR; produktiv wird intern getoggelt).

## Zustände

Geschlossen (nur Trigger-Badge) · offen (Menü erlaubter Übergänge, `role="listbox"`)
· disabled.

<Canvas of={StatusBadgeSelectStories.Variant_States} />

## Barrierefreiheit

- Trigger ist ein echtes `<button>` (Button-Atom) mit `aria-haspopup="listbox"`
  + `aria-expanded`.
- Menü ist `role="listbox"`, Optionen `role="option"` + `aria-selected`.
- Status-Bedeutung trägt das Label (StatusBadge), nicht nur die Tönung.
