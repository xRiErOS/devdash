# AssignDropZone

`status:stable` · Organism · Cluster `05.20 Actions` · OR-15 · GF-261 · Action „Drop something in Entity"

## Zweck

Drag-Ziel zum Zuweisen eines Issues an einen Sprint / eine Lane. Komponiert das
**ML-19 DropZone** (GF-276, I06) als Frame + over-Visual — kein eigenes Drag-Plumbing.

## Ebene-6-Capability-Gegencheck (A4)

OR-15 trägt **keine gemined Capability** (`_ebene6.md`) — präsentationaler Drag-Frame.
Die eigentliche Sprint-Zuweisung ist `issue-assign-sprint` (OR-16/OR-18-Domäne).

## Komposition + Scope

Das DropZone liefert den gestrichelten Rahmen + den (controlled) `active`-Zustand.
Die Issue-Payload-Verdrahtung (dataTransfer mit Issue-Key statt FileList) wickelt der
Consumer in der Realisierungsphase ab — die Story zeigt den **Vertrag** (Frame +
Zustände), nicht die DnD-Logik. Touch-Drag deferred (D09, Desktop-Fokus).

## Wann verwenden

- **Ja:** als Drop-Slot in einer SprintBoard-Lane (OR-14) oder am Sprint-Header.
- **Nein:** Datei-Upload → `AttachmentWidget` (OR-24, ebenfalls DropZone-basiert).
