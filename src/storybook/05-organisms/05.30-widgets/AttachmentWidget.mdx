# AttachmentWidget

`status:stable` · Organism · Cluster `05.30 Widgets` · OR-24 · GF-259 · V2-Rewrite GF-373

## Zweck

Zeigt die Anhänge einer Entität in der Terminal-Token-Sprache (EntityDetail V2): randlose
**CaptureWidget**-Shell + Liste (Name + Größe) + Upload-Button im create-Slot + **DropZone**
(ML-19, GF-276) für Drag&Drop. Klick auf einen Anhang schaltet den body auf eine In-Frame-
**Preview-Pane** um (Zurück / Öffnen / Löschen, D04) — kein Modal. Titellos (Titel zentral
im Accordion-Slot). Touch-Drop ist deferred (D09, Desktop-Fokus).

## Ebene-6-Capability-Gegencheck (A4)

OR-24 trägt **keine gemined Capability** (`_ebene6.md`) — reines Anhang-Display-Widget
(präsentational / Coverage-Lücke). Datei-Filter, MIME-/Magic-Byte-Prüfung und Upload
(Domäne) liegen beim Consumer (`onUpload`).

## Wann verwenden

- **Ja:** Anhang-Region einer Detail-Ansicht (Issue/Review-Feedback).
- **Nein:** Drag-Assign von Issues auf Sprints → `AssignDropZone` (OR-15, GF-261),
  ebenfalls DropZone-basiert.

## Props

<ArgTypes of={AttachStories} />

`attachments` = `{ key, name, size?, href?, kind?, previewUrl? }[]` (`kind:'image'` +
`previewUrl` → Bild-Preview, sonst Meta-Block). `uploadable` blendet DropZone + Upload-Button
ein, `removable` je Item + in der Preview einen Entfernen-Trigger. `onUpload(files)` bekommt
die rohe FileList; `onPick` öffnet den File-Dialog beim Consumer.

## Zustände

Liste + Upload + Entfernen · read-only · leer (DropZone bleibt, wenn uploadable) ·
Preview-Pane (Klick auf Anhang).

## Barrierefreiheit

- DropZone ist `role="button"`, `tabIndex=0`, mit `aria-label` + Enter/Space-Aktivierung.
- Anhang-Namen sind `Button`-Atome (Tastatur-Fokus) und öffnen die Preview; Entfernen und
  Zurück tragen sprechende `aria-label`; Öffnen ist ein externer `Link`.
